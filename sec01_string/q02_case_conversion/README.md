# 2. 대소문자 변환

### 1) 로직 흐름

1. 문자열을 문자 하나씩으로 분리한다.
2. 각 문자가 소문자인지, 대문자인지 검사한 후 (`isUpperCase()`) 소문자->대문자, 대문자->소문자로 변환하여 문자열에 추가한다. (`append`)

---

### 2) 핵심 개념

- 소문자, 대문자 구분 : `isUpperCase()`, `isLowerCase()`
- StringBuilder : `StringBuilder sb = new StringBuilder();` 로 사용하며, `append("문자열")`로 연결한다. `sb.toString()`을 통해 String으로 전환할 수 있다.
- (안 쓰고 +=로 붙여도 괜찮다. 하지만, 메모리 할당과 해제를 발생시켜 성능적으로 좋지는 않다. String 객체는 변경이 불가능하기에 문자열과 다른 문자열을 연결하면 새 문자열이 생성되고 이전 문자열은 가비지 컬렉터로 들어간다. 반면, StringBuilder은 기존 데이터에 더하는 방식으로 진행된다.)

* 추가 : 삼항연산자 사용 가능 `(조건문) ? (참일 때 값) : (거짓일 때 값);`
