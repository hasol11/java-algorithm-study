# 2. 보이는 학생

-> 선생님이 N명의 학생을 일렬로 세웠을 때, 맨 앞에 서 있는 선생님이 볼 수 있는 학생의 수를 구하는 프로그램 작성. (자신의 앞에 선 사람들보다 키가 커야 보인다.)

### 1) 로직 흐름

1.  **초기화**:
    - 맨 앞(0번 인덱스) 학생은 무조건 보이므로 `answer`는 1로 초기화한다.
    - 첫 번째 학생의 키를 `max` 변수에 담아 **'현재까지 가장 큰 키'**로 설정한다.
2.  **순회 (1번 인덱스부터)**:
    - 두 번째 학생(`i=1`)부터 끝까지 반복문을 돈다.
    - **비교**: 현재 학생(`arr[i]`)이 `max`보다 크다면?
      1.  선생님 눈에 보임 -> `answer` 증가 (`++`)
      2.  가장 큰 키 갱신 -> `max = arr[i]`
    - 작거나 같다면?
      - 앞사람에게 가려져 안 보이므로 넘어간다.

---

### 2) 핵심 개념

#### ① 시간 복잡도 최적화

- **비효율적 방법 (이중 for문)**:
  - 현재 학생(`i`)이 보일지 판단하기 위해 앞의 모든 학생(`0` ~ `i-1`)을 매번 다시 비교하면 연산 횟수가 너무 많아진다. ($N$이 클 경우 시간 초과 발생)
- **효율적 방법 (`max` 변수 활용)**:
  - 앞사람들을 일일이 비교할 필요 없이, **"지금까지 나온 사람 중 제일 키 큰 사람(`max`)"**보다 내가 큰지만 확인하면 된다.
  - 단 한 번의 순회(One-pass)로 문제를 해결할 수 있다.

#### ② 최댓값 갱신 알고리즘

- 배열을 순회하면서 조건에 만족하는 경우에만 최댓값(`max`)을 갱신하는 패턴을 자주 사용한다.

---

### 3. 트러블 슈팅 & 회고

이슈: O(N^2)의 이중 반복문 로직으로 접근했을 때 채점 시 시간 초과가 발생함.

원인 분석: 현재 학생보다 앞선 모든 학생을 매번 순회하며 비교하는 과정에서 중복 연산이 발생함.

개선: **'지금까지 본 학생 중 가장 큰 키'**만 기억하고 있으면 된다는 점에 착안하여, max 변수를 도입하고 단일 순회 로직으로 리팩토링함.
