# 3. 가위 바위 보

-> 각 회를 누가 이겼는지 출력하는 프로그램 작성

### 1) 로직 흐름

비김 처리 (Draw):

가장 먼저 if (A == B) 조건을 확인하여 비긴 경우(D)를 먼저 처리하고 넘긴다.

A 승리 처리:
A가 이기는 3가지 경우의 수를 `논리 연산자(||)`로 묶어서 한 번에 처리한다.
① 가위(1) vs 보(3)
② 바위(2) vs 가위(1)
③ 보(3) vs 바위(2)

B 승리 처리: 비기지도 않았고, A가 이기지도 않았다면 남은 경우는 무조건 B가 이긴 것이므로 else로 처리한다.

---

### 2) 핵심 개념

1. 논리 연산자 활용 (&&, ||)
   모든 경우의 수(9가지)를 일일이 if-else로 분기하는 대신, A가 이기는 조건들을 OR (||) 연산자로 묶어 코드의 가독성을 높인다.

2. 분기 처리 전략
   특수한 경우(비김) -> 한 쪽 승리(A) -> 나머지(B) 순서로 조건을 배치하여 불필요한 연산을 줄인다.

---

### 3. 트러블 슈팅 & 회고

문제점

처음에는 사람이 생각하는 방식대로 "A가 가위를 냈을 때 B가 바위면... 보자기면..." 처럼 모든 상황을 Nested If문과 삼항 연산자로 나열하여 코드가 복잡하고 지저분해졌다.

해결

"상황(Story)"을 나열하는 것이 아니라 "결과(Result)"를 기준으로 조건을 분류해야 함을 깨달았다.
A가 이기는 경우만 명확히 정의하면, 나머지는 자동으로 B의 승리가 된다는 점을 이용해 로직을 단순화했다.
