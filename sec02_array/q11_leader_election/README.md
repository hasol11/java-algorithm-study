# 11. 임시반장 정하기

-> N명의 학생들에 대해 1학년부터 5학년까지의 반 기록이 주어질 때, 한 번이라도 같은 반이었던 적이 가장 많은 학생을 임시 반장으로 정하는 프로그램

### 1) 로직 흐름

1. 데이터 입력 (2차원 배열): 1-based 인덱스를 활용하여 arr[N+1][6] 크기의 2차원 배열을 선언하고, 1번 학생부터 N번 학생까지의 1~5학년 반 데이터를 입력받는다.

2. 3중 for문: - 1번 루프(i): 기준이 되는 학생 (반장 후보)
   2번 루프(j): 비교 대상이 되는 학생
   3번 루프(k): 1학년부터 5학년까지 순회하며 같은 반인지 확인

3. 중복 카운트 방지: arr[i][k] == arr[j][k] 조건이 참이라면 카운트(cnt)를 1 증가시키고, 반드시 break를 호출하여 해당 학생(j)과의 나머지 학년 비교를 건너뛴다.

4. 최댓값 및 반장 갱신: 기준 학생(i)의 최종 카운트가 현재 최댓값(max)보다 엄격히 클 때만(>) 최댓값과 정답(answer) 번호를 갱신한다.

---

### 2) 오답 정리

중복 처리 누락 (break의 부재): 한 학생과 1학년, 2학년 때 모두 같은 반이었다고 해서 카운트를 2번 올리면 안 된다. 문제의 조건은 '같은 반이었던 학생의 수'를 구하는 것이다.

해결: 같은 반인 것을 한 번이라도 확인했다면 break문을 사용해 루프를 탈출하고 다음 학생 비교로 넘어가야 한다.
