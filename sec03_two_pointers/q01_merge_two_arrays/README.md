# 1. 두 배열 합치기

-> 두 배열을 합쳐 하나의 오름차순 정렬된 배열을 만들어내는 프로그램

### 1) 로직 흐름

1. a 배열에 P1, b 배열에 P2 각각 포인터가 존재하고, 0번 인덱스를 가리키고 있다.
2. a[P1]과 b[P2]를 비교해서 answer라는 arraylist에 추가한다.
3. 비교해서 더 작은 배열의 내용을 추가하고 포인터를 옮긴다.
4. 두 배열의 길이가 다르거나 데이터 분포에 따라 어느 한쪽 포인터가 먼저 끝에 도달하게 되면, 남아있는 배열의 나머지 원소들을 순서대로 추가해준다.

### 2) 핵심 개념

- 일반 정렬 사용 시: $O((N+M) \log (N+M))$
  - 만약 $N+M = 60,000$ 이라면, 약 $60,000 \times 16 = 960,000$ 번의 연산이 필요하다.
- 투 포인터 알고리즘 사용 시: $O(N+M)$
  - 두 배열의 원소를 딱 한 번씩만 훑고 지나가므로, 약 $60,000$ 번의 연산으로 끝난다.- 결론: 이미 정렬되어 있는 배열들을 합칠 때는 투 포인터가 압도적으로 빠르다!
